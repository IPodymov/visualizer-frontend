# Архитектура проекта

Проект построен с использованием методологии **Feature-Sliced Design (FSD)**. Это обеспечивает масштабируемость и поддерживаемость кодовой базы за счет разделения функциональности на слои и слайсы.

## Структура слоев

Код организован в следующей иерархии (от верхнего уровня к нижнему):

1.  **App (`src/app`)**
    *   Инициализирующая логика приложения.
    *   Глобальные стили, провайдеры (Redux, Router) и настройки.

2.  **Pages (`src/pages`)**
    *   Композиция полноценных страниц из виджетов, фич и сущностей.
    *   Примеры: Главная страница (`home-page`), Страница сравнения (`compare-page`), Страница авторизации (`login-page`).

3.  **Widgets (`src/widgets`)**
    *   Крупные функциональные блоки, объединяющие несколько фич и сущностей.
    *   Примеры: `Header` (шапка сайта), `ComputingLandscape` (виджет визуализации).

4.  **Features (`src/features`)**
    *   Бизнес-ценные действия пользователя.
    *   Примеры: `Auth` (авторизация/регистрация), `PlanComparison` (логика сравнения планов).

5.  **Entities (`src/entities`)**
    *   Бизнес-сущности, с которыми работает приложение.
    *   Примеры: `Plan` (учебный план), `Session` (сессия пользователя).
    *   Содержат модели данных, UI-компоненты сущности (например, карточка плана).

6.  **Shared (`src/shared`)**
    *   Переиспользуемый код, не привязанный к конкретной бизнес-логике.
    *   UI-компоненты (кнопки, инпуты), утилиты, API-клиенты, конфиги.

## Принципы взаимодействия

*   Модули могут импортировать функциональность только из нижележащих слоев.
*   Например, `widgets` может использовать `features`, `entities` и `shared`, но не может использовать `pages`.
*   Слои `app` и `pages` являются точками сборки приложения.

## Используемые Технологии

*   **React** + **TypeScript**
*   **Vite** (сборщик)
*   **Redux Toolkit** (управление состоянием и RTK Query для запросов)
*   **React Router** (маршрутизация)
*   **CSS Modules / Global CSS** (стилизация)
